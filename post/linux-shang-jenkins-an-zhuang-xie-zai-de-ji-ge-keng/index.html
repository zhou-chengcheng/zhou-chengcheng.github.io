Linux 上 Jenkins安装、卸载的几个坑

2019-05-20

下载:
wget https://pkg.jenkins.io/redhat/jenkins-2.178-1.1.noarch.rpm

安装：
rpm -ivh jenkins-2.83-1.1.noarch.rpm

配置Jenkins：
vim /etc/sysconfig/jenkins

修改端口:
JENKINS_PORT="8888"

启动服务:
systemctl start jenkins

访问： 略

如果正常操作是没有问题的，但是如果忘记密码，或者什么原因要卸载重新安装的话就会有一些坑

rpm卸载
rpm -e jenkins

检查是否卸载成功
rpm -ql jenkins

彻底删除残留文件：
find / -iname jenkins | xargs -n 1000 rm -rf

修改配置文件 ( 注意： 卸载后一定要操作这一步，不然会一直启动Jenkins失败) :
vim /etc/init.d/jenkins

下面的Java路径换成自己的路径：/usr/java/jdk1.8.0_161

candidates="
/usr/java/jdk1.8.0_161/bin/java
/usr/java/jdk1.8.0_161/jre/bin/java
"
